🐛 Bug Fix: xmlPatch.ts::patchXmlBlock silent-drop 수정

Symptom. CreateDomain에 fixed_values를 전달해도 SE11에서 고정값이 비어 있음. update 단계가 실패 없이 silent drop.

Root cause. fr0ster의 patchXmlBlock은 대상 태그가 XML에 존재하면 교체하지만, 없을 때는 xml.replace가 no-op을 반환 → 새로 생성된 skeleton 도메인에 <doma:fixValues> 태그가 아예 없을 때 주입 실패.

❌ Before (v3.13.0 upstream)
function patchXmlBlock(xml, tag, content) { const regex = new RegExp( `<${tag}[\\s\\S]*?</${tag}>|<${tag}\\s*/>` ); return xml.replace(regex, content); // ↑ 매치 실패 시 원본 반환 = silent drop }
✅ After (@babamba2 fork)
function patchXmlBlock(xml, tag, content) { const regex = new RegExp( `<${tag}[\\s\\S]*?</${tag}>|<${tag}\\s*/>` ); if (regex.test(xml)) return xml.replace(regex, content); // 태그 부재 시 root 닫는 태그 앞에 삽입 const root = xml.match(/<\/[a-zA-Z0-9:_-]+>\s*$/); if (!root) return `${xml}\n${content}`; return xml.slice(0, root.index) + content + '\n' + xml.slice(root.index); }
